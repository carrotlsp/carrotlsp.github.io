day02-结构体

一、结构体

结构体（Structure）在C语言中是一种非常重要的数据结构，它允许你将多种不同类型的数据项（称为成员）组合成一个单一类型。

这使得结构体非常适合用来创建复杂的数据模型，结构体的使用可以大大提高程序的清晰度和易维护性。

二、内存对齐

1、为什么要内存对齐？

不同类型的数据访问速度最优化时，其地址往往需要满足一定的对齐要求。

例如，某些硬件平台上，访问4字节整数（如 int 类型）的地址可能需要是4的倍数。如果数据被放置在满足这些要求的地址上，CPU的数据访问速度就可以最大化。

2、如何进行内存对齐？

  • 对齐规则：编译器将按照结构体中最宽基本类型成员的对齐要求，为整个结构体选择一个“对齐因子”（也称为“对齐量”）。整个结构体的起始地址会按照这个对齐因子来对齐。结构体内的每个成员也会根据其类型的对齐要求，在结构体内部相对起始地址进行对齐。
  • **填充字节（Padding）:**为了满足内存对齐的要求，编译器可能在结构体的成员之间或结构体的末尾拖入未使用的空间（称为“填充字节”），这可以确保每个成员都在对其对齐要求的地址上开始。

3、举例说明？

  • a 直接放在结构体的开始。

  • 在 a 和 b 之间插入3个填充字节，以确保 b 的地址是4的倍数，满足 int 类型的对齐要求。

  • c 紧跟在 b 后面，但此时结构体的总大小为9字节。为了使结构体数组中每个元素的地址都满足最严格的对齐要求（在这里是4字节对齐），在结构体的末尾还会添加3个填充字节，使得整个结构体的大小变为12字节。

4、如何优化上面的结构体？

  • 了解内存对齐可以帮助你更好地设计结构体以减少内存空间的浪费。
  • 一个常见的技巧是按照 从最宽到最窄 的成员顺序声明结构体成员，这样可以减少填充字节的数量。
